# AWS IPs

[![Build Status](https://travis-ci.org/kreuzwerker/awsips.svg?branch=master)](https://travis-ci.org/kreuzwerker/awsips)

Commandline utility (`awsips`) to [query](https://ip-ranges.amazonaws.com/ip-ranges.json) IP addresses used by Amazon Web Services. It can also be used to build [CloudFormation](http://aws.amazon.com/cloudformation/) stacks containing Security Groups referencing such IPs.

The tool has to be used in one of several modes (`-m` flag):

* `regions`: returns a JSON array over all supported regions
* `services`: returns a JSON array over all supported services
* `list-files`: generates JSON files (in `$PWD`) for every region and service, containing a JSON array of IP addresses in CIDR notation
* `list-tree`:returns a JSON object mapping regions to services, and services to an array of IP addresses in CIDR notation
* `list-stack`: returns the IP addresses as Security Groups defined in a CloudFormation stack (in JSON)

When using a list mode, the resulting output can be filtered by:

* region or multiple, comma-seperated regions (`-rf` flag)
* service or multiple, comma-seperated services (`-sf` flag)

When using stack mode, the following additional parameters are available:

* port or multiple, comma-seperated ports (`-p` flag) - when omitted the default is to open ports from 1 to 65535

The JSON (including the IP addresses) emitted by `awsip` is always sorted lexicographically to make diffing and checksum generation easier.

When access to individual elements returned by a modes is desired, an external tool like [jq](https://github.com/stedolan/jq) is recommended.

``` 
aws cloudformation create-stack --stack-name test --template-body file://stack.json --parameters ParameterKey=vpc,ParameterValue=vpc-12345678
```
